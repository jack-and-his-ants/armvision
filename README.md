# ARMVision: SIMD-Accelerated Image Processing Engine

[![Language](https://img.shields.io/badge/Language-C11-blue.svg)](https://en.wikipedia.org/wiki/C11_(C_standard_revision))
[![Platform](https://img.shields.io/badge/Platform-ARM64%20(AArch64)-red.svg)](https://www.arm.com/)
[![Build](https://img.shields.io/badge/Build-CMake-orange.svg)](https://cmake.org/)

**ARMVision** is a high-performance library and CLI tool for low-level BMP image processing, optimized for the **ARM64 (AArch64)** architecture. The project demonstrates the power of **SIMD (NEON)** instructions and hand-written **Assembly** kernels to break performance bottlenecks imposed by standard compilers (even at `-O3` optimization levels).



## Key Features

- **Performance-First Design**: Optimized for ARMv8-A architecture (Apple Silicon, Raspberry Pi 4/5, AWS Graviton).
- **SIMD Acceleration**: Full utilization of **ARM NEON** units for parallel pixel processing.
- **Hand-written Assembly Kernels**: Critical compute paths written in pure AArch64 Assembly for total control over register allocation and instruction pipelining.
- **Professional CLI**: POSIX-compliant command-line interface (`getopt`) supporting dynamic kernel sizing and channel selection.
- **Benchmarking Suite**: Built-in micro-benchmarking tool measuring execution time, featuring iterations to eliminate noise.

## Supported Filters

| Filter | Implementations | Description |
| :--- | :--- | :--- |
| **Negative** | C / ASM / **NEON** | Bitwise color inversion using the `mvn` vector instruction. |
| **Grayscale** | C / ASM / **NEON** | High-speed luma conversion using fixed-point arithmetic. |
| **Monochrome** | C / ASM / **NEON** | Single-channel isolation using SIMD bit-masking. |
| **Gaussian Blur**| C | Smooth blurring using the 3-sigma rule ($\sigma = \text{size}/6$). |
| **Sobel** | C | Edge detection via 3x3 convolution kernels. |
| **Box Blur** | C | Mean filtering with user-defined kernel dimensions. |

## Performance Benchmarks

Tests conducted on **ARM64** hardware using `gcc -O3 -march=native`.  
*The baseline represents C code with the highest compiler optimization level enabled.
A bitmap used for this benchmark is 256x256 pixels and the filter was applied 1000 times. Tested on Apple M2 (AArch64) 8GB RAM*

| Filter | C (-O3) | ASM (Scalar) | **ASM (NEON)** | **Speedup** |
| :--- |:--------|:-------------|:---------------|:------------|
| **Negative** | 0.14s   | 0.073s       | **0.009s**     | **~16.3x**  |
| **Monochrome** | 0.078s  | 0.042s       | **0.009s**     | **~9.09x**  |
| **Grayscale** | 0.13s   | 0.098s       | **0.012s**     | **~10.8x**  |

> **Engineering Insight:** The NEON implementation outperforms `-O3` auto-vectorization primarily by utilizing **Structured Loads** (`ld3`). This hardware-level de-interleaving separates BGR triplets into distinct registers, eliminating the heavy shuffle overhead typically generated by compilers.
## Memory Stability
The engine was stress-tested using the Apple `leaks` utility during high-frequency processing cycles.
- **Total Allocations**: Verified for every execution path.
- **Result**: **0 leaks / 0 bytes leaked**.
## Usage Instructions

### Build
```bash
mkdir build && cd build
cmake ..
make
```
### CLI Examples
```bash
# Basic filtering (NEON mode)
./armvision -i input.bmp -o output.bmp -f grayscale -m neon

# Gaussian Blur with a 7x7 kernel
./armvision -i input.bmp -o blurred.bmp -f gaussian_blur -s 7

# Red channel isolation (Monochrome)
./armvision -i input.bmp -o red.bmp -f monochrome -c r -m neon

# Run benchmark (500 iterations)
./armvision -i input.bmp -f grayscale -m bench -n 500
```
